State management quiz

1. Can post requests be bookmarked? What are the problems?
No, a post request can't be bookmarked. Trying to do so will result in a get request. The problem is just because a post request is not idempotent. Because, normally when you bookmarked a request, you do that in the objective to retrieve the same thing later, but it can't be like that for a post request because it's not idempotent.
2. What is the purpose of request dispatching?
The purpose of request dispatching is to allow interaction between servlet and JSP.
3. What is the difference between redirect and request dispatch?
A request dispatch is operated only to the side of the server and also you can dispatch a request only to a resource that is on the same server.
Whereas a redirect is doing to the client side and you can redirect even to a resource that is not on the same server.
4. What is an attribute ?
An attribute is a variable that you can save in one of the scope of an application. When I say scope, I mean Page, Request, Session, Application. To save the attribute you call the setAttribute() method and to retrieve you call the getAttribute() method
5. What is the difference between attributes and parameters?
There are 2 main differences between attributes and parameters:
There first one is: only the scope Request can contain parameter. Whereas an attribute can be saved in the Page, Request, Session or Application scope.
The second one is: an attribute is passed by the user using the setAttribute() method whereas the parameter is taken from the url or the request body and saved automatically by the web container in the request object.
6. What are dangers of using attributes?
The dangers of using attributes are to know in which scope to save them. For example, if you save a confidential attribute in the application scope, it will exist during the lifetime of the application and it will be visible to all the users.
7. What does it mean to say that http is stateless? Give an example of a stateful protocol.
Saying that http is a stateless protocol means it doesn't keep anything and doens't have any idea about its previous state. It execute only once and after that it looses all information about itself.
