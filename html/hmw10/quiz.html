JSP Introduction Quiz

1. What is the main value of Java Server Pages?
The Java Server Pages generally called JSP allows to decouple the view from the servlet(controller). It allows to use the MVC pattern properly and also allows to use dynamic data within html.
2. How are JSP pages related to servlets?
JSP pages are servlets. When you run your application, the web container is going to compile JSP pages into servlet.
3. How are JSP pages related to HTML?
JSP pages is done mostly with HTML code. Besides, its main role is to contain HTML and to allow the use of dynamic java code.
4. Give an example of a JSP scriptlet and show how it will look in the JSP servlet .
Ex : <% out.print("Moreno"); %>
In the JSP servlet it will look like that:
PrintWriter out= response.getWriter();
out.print("Moreno");
5. Give an example of a JSP declaration and show how it will look in the JSP servlet.
Ex: <%! int count= 0; %>
In the servlet it will look almost the same:
int count= 0;
6. Give an example of a JSP comment and show how it will look in the JSP servlet.
Ex: <%--My comment --%>
In the servlet it will look like that:
//My comment
or
/*My comment*/
7. Give an example of a JSP expression and show how it will look in the JSP servlet.
We suppose that count is already declared in a scriptlet.
Ex: <%= count++ %> In the servlet it will look almost the same:
int count= 0; count++;
8. Give an example of a JSP directive and show how it will look in the JSP servlet.
Ex: <%@ page import="java.util.Date" %>
In the servlet it will look like that:
import java.util.Date;
9. Explain how an EL expression is evaluated.
When you have an EL expression like that ${person.age} or ${person[age]}, if person is a bean, age is a property of person, if person is a map, age is a key.
10. Explain how servlet attributes are involved in EL expressions.
When you have an El expression, the web container checks firstly to see if the attribute is available in the page scope, if it is, it picks it, in the opposite it checks in the request scope, if it is it picks it in the opposite it checks in the session scope, if it is it picks it, in the opposite it checks in the application scope, if it is, it picks it, in the opposite it just gives it up.
11. Explain how servlets and JSPs use request dispatch to interact.
To interact between them we can use the method getRequestDispatcher() of the HttpServletRequest that will allow you to forward the request and the response object to the servlet.
N.B. This forwarding is completely different from the redirect, because the redirect is done to the client side whereas the forwarding is done to the server side.
